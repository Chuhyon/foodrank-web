## 5. 채널 신뢰도 표시 규칙 (●○○○○)

```
source_count = 해당 키워드에 데이터가 있는 채널 수 (최대 4)
표시 방식   = 5칸 중 source_count개를 채운 도트(●)로 표시

표시 규칙:
  4개 채널 → ●●●●○  (4/5 — 최고 신뢰도이나 의도적으로 만점 미표시)
  3개 채널 → ●●●○○
  2개 채널 → ●●○○○
  1개 채널 → ●○○○○  (단일 채널, 낮은 신뢰도)

5칸을 사용하는 이유:
  집계 채널이 4개이지만 5칸으로 표시하면 "더 많은 채널로 검증될 수 있다"는
  시각적 여백을 제공합니다. 4칸 만점(●●●●) 표시 시 사용자가 항상 만점으로
  인식해 신뢰도 차별화가 어려워집니다.
```

```jsx
// components/SourceIndicator.jsx
export default function SourceIndicator({ count }) {
  const MAX = 5;  // 시각적 스케일: 5칸 중 source_count개 채움 (채널 수는 최대 4)
  return (
    <span className="source-indicator" title={`${count}개 채널 집계`}>
      {Array.from({ length: MAX }, (_, i) => (
        <span key={i} className={i < count ? 'dot filled' : 'dot empty'}>
          {i < count ? '●' : '○'}
        </span>
      ))}
    </span>
  );
}
```

## 6. 업데이트 시각 표시 정책

```
표시 형식: "데이터 기준: YYYY.MM.DD HH:00 수집"

규칙:
  - Supabase keyword_trends.collected_at 필드값 사용
  - 분 단위 반올림 없이 시(HH):00 으로 표시
  - 데이터 수집 실패 시 "(이전 데이터)" 접미사 추가
    예: "데이터 기준: 2025.03.09 12:00 수집 (이전 데이터)"

ISR 캐싱(10분)과의 관계:
  - ISR은 Next.js 서버 렌더링 캐시일 뿐
  - 사용자에게 노출되는 "데이터 기준" 시각은 DB 수집 시각 기준
  - "10분마다 새로고침"이 아님을 UI에서 혼동 유발하지 않도록 주의
```
